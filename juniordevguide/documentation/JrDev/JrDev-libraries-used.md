# Libraries Used in the Application

This document provides a list of libraries and frameworks utilized in the application, along with their respective versions and purposes.

## Frontend Libraries

1. **React**
   - **Version:** 17.0.2
   - **Purpose:** A JavaScript library for building user interfaces, allowing for the creation of reusable UI components.

2. **Redux**
   - **Version:** 4.1.0
   - **Purpose:** A state management library for JavaScript applications, enabling predictable state management across the application.

3. **Axios**
   - **Version:** 0.21.1
   - **Purpose:** A promise-based HTTP client for the browser and Node.js, used for making API requests.

4. **React Router**
   - **Version:** 5.2.0
   - **Purpose:** A library for routing in React applications, allowing for navigation between different components.

## Backend Libraries

1. **Express**
   - **Version:** 4.17.1
   - **Purpose:** A web application framework for Node.js, designed for building web applications and APIs.

2. **Mongoose**
   - **Version:** 5.10.9
   - **Purpose:** An ODM (Object Data Modeling) library for MongoDB and Node.js, providing a schema-based solution to model application data.

3. **jsonwebtoken**
   - **Version:** 8.5.1
   - **Purpose:** A library for creating and verifying JSON Web Tokens (JWT), used for authentication and authorization.

4. **dotenv**
   - **Version:** 8.2.0
   - **Purpose:** A zero-dependency module that loads environment variables from a `.env` file into `process.env`.

## Testing Libraries

1. **Jest**
   - **Version:** 26.6.0
   - **Purpose:** A delightful JavaScript testing framework with a focus on simplicity, used for unit and integration testing.

2. **Supertest**
   - **Version:** 6.1.0
   - **Purpose:** A library for testing HTTP servers in Node.js, often used in conjunction with Jest for API testing.

## Development Tools

1. **ESLint**
   - **Version:** 7.14.0
   - **Purpose:** A static code analysis tool for identifying problematic patterns in JavaScript code, helping maintain code quality.

2. **Prettier**
   - **Version:** 2.2.1
   - **Purpose:** An opinionated code formatter that enforces a consistent style across the codebase.

## Conclusion

This document serves as a reference for the libraries and frameworks used in the application. Understanding these tools will help in maintaining and developing the codebase effectively.
